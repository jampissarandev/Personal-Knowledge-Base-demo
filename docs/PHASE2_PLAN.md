**Conventions to follow (match the existing codebase, do not invent new patterns):**

- File-scoped namespaces, C# 13, `var` for locals, explicit types for public APIs.
- DTOs: positional `record` for responses, init-only `record` for requests.
- Validation: `DataAnnotations` (`[Required]`, `[StringLength]`, `[MaxLength]`) — same as
  `Auth/DTOs/*.cs`.
- Error envelope: reuse `ErrorResponse.Of(code, message, details?)` — same as `AuthController`.
- Ownership: every read/write must filter by `UserId == currentUserId`. No
  `FindByIdAsync` without an ownership check. This is OWASP A01 territory.
- Service pattern: services take `AppDbContext` via DI; controllers take the service interface.
  Mirror the `AuthService` / `IAuthService` split.
- Mapping: use `Mapster` (already in csproj). Centralise in `Mappings/MapsterConfig.cs`; call
  `MapsterConfig.Register()` from `Program.cs` after `builder.Build()` is NOT correct — it must
  run before app build, so wire it into `Program.cs` near the top, similar to how `Log.Logger`
  is initialised.
- Logging: follow the existing structured pattern
  (`_logger.LogInformation("event={Event} userId={UserId} noteId={NoteId}", ...)`). New event
  names: `note_created`, `note_updated`, `note_deleted`, `note_pin_toggled`, `tag_created`,
  `tag_deleted`, `folder_created`, `folder_deleted`.

**Endpoint-specific contracts (binding for the implementer):**

- `GET /api/notes?folderId=&tagId=&isPinned=` — returns `200 NoteResponse[]`.
  All three filters optional, AND-combined. `isPinned` accepts `true|false|1|0`.
- `GET /api/notes/{id}` — `200 NoteResponse` or `404 NOTE_NOT_FOUND`.
- `POST /api/notes` — `201 NoteResponse` with `Location: /api/notes/{id}`; `400` on validation;
  `404 FOLDER_NOT_FOUND` / `404 TAG_NOT_FOUND` if any referenced id is wrong or owned by
  another user.
- `PUT /api/notes/{id}` — `200 NoteResponse`; `404 NOTE_NOT_FOUND`; `400` on validation;
  tag list is **replaced atomically** (delete missing + insert new), so omit `tagIds` to keep
  existing.
- `DELETE /api/notes/{id}` — `204`; `404 NOTE_NOT_FOUND`. Cascade deletes `NoteTag` rows.
- `PATCH /api/notes/{id}/pin` — `200 NoteResponse` with toggled `isPinned`.
- `GET /api/tags` — `200 TagResponse[]` ordered by `Name`.
- `POST /api/tags` — `201 TagResponse`; `409 TAG_EXISTS` on duplicate `(UserId, Name)`
  (trim + case-insensitive compare).
- `DELETE /api/tags/{id}` — `204`; `404 TAG_NOT_FOUND`. Cascade deletes `NoteTag` rows.
- `GET /api/folders` — `200 FolderResponse[]` ordered by `Name`; include `NoteCount`.
- `POST /api/folders` — `201 FolderResponse`; `409 FOLDER_EXISTS` on duplicate.
- `DELETE /api/folders/{id}` — `204`; `404 FOLDER_NOT_FOUND`. Schema already
  `SetNull` on `Note.FolderId`, so notes become unfiled automatically.

**NoteResponse shape (binding):**

```csharp
public record NoteResponse(
    Guid Id,
    string Title,
    string ContentJson,
    string ContentText,
    bool IsPinned,
    Guid? FolderId,
    string? FolderName,
    IReadOnlyList<TagRef> Tags,        // [{Id, Name}]
    DateTime CreatedAt,
    DateTime UpdatedAt);

public record TagRef(Guid Id, string Name);
```

`ContentText` is required on write (controller derives it from `ContentJson` if not supplied,
plain-text projection; documented in the create/update DTOs). No FTS5 wiring in this phase
(Phase 3's job).

### Critical Findings — FIXED IN PHASE

**C-1 [Security+Code]: Tag/Folder case-insensitive uniqueness race + EF translation error**

- **Where:** `TagService.cs:39`, `FolderService.cs:43`, `AppDbContext.cs:63-82`
- **Issue:** The application used `Name.ToLowerInvariant()` in the existence check, which (a) is
  not translatable to SQL by EF Core/SQLite (caused **500 Internal Server Error** on
  `POST /api/tags` and `POST /api/folders` — discovered by QA verifier) and (b) leaves a race
  window where two concurrent requests with different casing could both pass the app check
  and bypass the case-sensitive SQLite unique index.
- **Fix applied:**
  1. Added `entity.Property(...).UseCollation("NOCASE")` on `Tag.Name` and `Folder.Name` in
     `AppDbContext.OnModelCreating` so the unique index `(UserId, Name)` is
     case-insensitive at the DB level.
  2. Replaced the LINQ `Name.ToLowerInvariant() == normalized` predicate with
     `Name == trimmedName` — the column's `NOCASE` collation now handles case-folding.
  3. Generated EF migration `20260619151755_TagFolderNameNocase` and applied via
     `dotnet ef database update`.
  4. QA confirmed `POST /api/tags {name:"TEST-TAG-X"}` after creating `"test-tag-x"` returns
     **409** (race closed; test in QA log: `[4] Duplicate tag (NOCASE): PASS`).

### Important Findings — FIXED IN PHASE

**I-1 [Security]: `FolderService.ListAsync` `NoteCount` missing `UserId` filter**
- **Fix:** Added `&& n.UserId == userId` to the Count predicate at `FolderService.cs:29`.
  Defense-in-depth: even if a future code change allowed cross-tenant folder references,
  the count would still be scoped.

**I-2 [Code]: Validation error format inconsistency (ProblemDetails vs ErrorResponse)**
- **Fix:** Added `ApiBehaviorOptions.InvalidModelStateResponseFactory` in `Program.cs:36-50`
  that converts ModelState errors to `ErrorResponse.Of("VALIDATION_ERROR", ...)` envelope.
  QA confirmed: empty tag name and empty note title both return `400` with `VALIDATION_ERROR`
  body.

**I-3 [Code]: `MapsterConfig` `Note → NoteResponse` depends on eager-loaded navigations**
- **Status:** Documented by code-reviewer as a maintenance hazard. Currently safe because all
  call sites that invoke `.Adapt<NoteResponse>()` include the necessary `.Include()` chains.
  Not blocking; deferred to follow-up.

**I-4 [Code]: `NoteService.UpdateAsync` generates N individual INSERTs for tags**
- **Status:** Suboptimal but tolerable for typical tag counts (< 20). Deferred to follow-up.
